# Contribuindo

Obrigado pelo interesse em contribuir com este projeto.

## Antes de começar

- Verifique se já existe uma issue aberta para o problema ou sugestão.
- Para mudanças maiores, abra uma issue antes de implementar para alinhar escopo e direção.
- Mantenha o foco em melhorias objetivas, legíveis e consistentes com a proposta visual do projeto.

## Como contribuir

1. Faça um fork do repositório.
2. Crie uma branch descritiva a partir da `main`.
3. Implemente a mudança com escopo claro.
4. Teste localmente antes de abrir o pull request.
5. Envie um pull request com contexto suficiente para revisão.

## Padrões esperados

- Preserve a proposta visual e a identidade do projeto.
- Priorize responsividade, acessibilidade básica e boa legibilidade.
- Evite mudanças amplas sem necessidade.
- Mantenha nomes e estrutura simples de entender.

## Checklist para pull request

- A mudança resolve um problema real ou melhora claramente o projeto.
- O impacto visual foi revisado em desktop e mobile.
- Não há regressões óbvias de navegação ou conteúdo.
- O README e a documentação foram atualizados se necessário.

## Reportando problemas

Ao abrir uma issue, informe:

- contexto do problema
- comportamento atual
- comportamento esperado
- passos para reproduzir
- prints ou gravações, quando ajudarem

## Comunicação

Se houver dúvida sobre direção, arquitetura ou UX, prefira alinhar antes de investir em implementação extensa.
